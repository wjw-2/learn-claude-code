# Security Audit

提交前必须进行安全检查。

## 强制检查项

### 密钥管理
- [ ] 源码中无硬编码密钥/API Key/密码/Token
- [ ] 使用环境变量或密钥管理器
- [ ] `.env` 在 `.gitignore` 中
- [ ] 启动时验证必需密钥存在

### 输入验证
- [ ] 所有用户输入在边界验证
- [ ] 使用 schema 验证（Pydantic 等）
- [ ] 快速失败，清晰的错误信息
- [ ] 不信任外部数据（API 响应、用户上传）

### 注入防护
- [ ] SQL 使用参数化查询（不拼接字符串）
- [ ] HTML 输出转义（防 XSS）
- [ ] 文件路径 sanitization（防路径穿越）
- [ ] 命令注入防护（不直接拼接 shell 命令）

### 认证与授权
- [ ] 所有端点有认证检查
- [ ] 权限检查在业务逻辑层
- [ ] Token 安全存储和传输（HTTPS）
- [ ] 会话超时和刷新机制

### 错误处理
- [ ] 错误信息不泄露内部细节
- [ ] 不返回堆栈给用户
- [ ] 服务端记录详细错误日志
- [ ] 异常不中断服务

### 速率限制
- [ ] 所有公开端点有速率限制
- [ ] 登录/注册有额外的频率控制

## 发现安全问题的处理

1. **立即停止** 当前工作
2. 用安全审计工具扫描（bandit for Python）
3. 修复 CRITICAL 问题再继续
4. 如有泄露的密钥，立即轮换
5. 检查代码库是否有类似问题

## Python 安全扫描

```bash
bandit -r .
```
