# Code Review

代码提交前必须进行审查。

## 审查时机

- 编写或修改代码后立即审查
- 涉及认证、支付、用户数据时重点审查安全
- 合并到主分支前必须审查

## 审查清单

### 安全性 (CRITICAL - 必须修复)
- [ ] 无硬编码密钥、密码、Token
- [ ] 用户输入全部经过验证
- [ ] SQL 查询使用参数化（防注入）
- [ ] 无 XSS 漏洞（HTML 已转义）
- [ ] 错误信息不泄露敏感数据

### 代码质量 (HIGH - 应该修复)
- [ ] 函数聚焦（<50 行）
- [ ] 文件内聚（<800 行）
- [ ] 无深层嵌套（>4 层用早返回）
- [ ] 错误处理明确（不吞异常）
- [ ] 变量/函数命名清晰

### 性能 (MEDIUM - 建议优化)
- [ ] 无 N+1 查询
- [ ] 大数据集有分页/限制
- [ ] 缓存了昂贵操作

## 审查流程

1. `git diff` 查看变更
2. 先检查安全性清单
3. 再检查代码质量清单
4. 运行相关测试
5. 标记问题严重程度：CRITICAL / HIGH / MEDIUM / LOW
6. CRITICAL 和 HIGH 必须修复后才能合并
