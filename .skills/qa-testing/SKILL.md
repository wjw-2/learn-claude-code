# QA Testing

验证应用是否正常工作——不只是测试通过。

## 测试类型

1. **冒烟测试**：核心功能能否走通
2. **边界测试**：空输入、超长输入、特殊字符
3. **错误恢复**：网络断开、超时后能否恢复
4. **回归测试**：新功能不影响已有功能

## 测试流程

1. 启动开发服务器
2. 导航到目标路由/功能
3. 验证：正常路径 → 边界情况 → 控制台错误 → 网络失败
4. 截图/录屏保存证据到 `qa-evidence/`
5. 问题分级：
   - **P0**: 阻塞发布（核心功能不可用）
   - **P1**: 合并前修复（重要功能有缺陷）
   - **P2**: 记录待修（体验问题）

## 注意事项
- "测试通过" ≠ QA 通过。测试验证代码路径，QA 验证用户体验
- 每次功能变更后至少跑一次冒烟测试
- 记录浏览器控制台错误和网络请求失败
