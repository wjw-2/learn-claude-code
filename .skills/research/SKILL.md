# Research

实现前先调研，避免重复造轮子和过时的方案。

## 何时触发

- 实现 >200 行的新功能前（找参考架构）
- 选择库/框架/方案前（当前最佳实践）
- 需要验证某个事实前（模型版本、API 变更）
- 用户问了代码库之外的问题

## 研究策略

1. **GitHub 优先**: `gh search repos` 和 `gh search code` 找现有实现
2. **官方文档**: 用 Context7 或官方文档确认 API 行为
3. **Exa 搜索**: 更广泛的代码示例和技术文章
4. **代码库内部**: grep 现有实现作为参考

## 决策框架

**如果要用训练数据断言可能过时的事实，先搜索。**

特别适用于：
- 模型名称和版本（几个月就过时）
- 库的 API 和最佳实践（大版本会变）
- 价格、可用性、功能对比
- 安全公告、CVE、废弃通知

```
## 方案对比
| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| A | ... | ... | ... |
| B | ... | ... | ... |

## 推荐
[推荐方案 + 理由]

## 参考链接
- [资源 1](url)
- [资源 2](url)
```
