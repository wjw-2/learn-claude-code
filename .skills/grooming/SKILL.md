# Grooming

战略需求管理和优先级排序。

## 模式

| 模式 | 用途 |
|------|------|
| **explore** | 并行调查 → 综合主题 → 优先级需求列表 |
| **rethink** | 深度架构探索 → 一个明确推荐 |
| **scaffold** | 项目初始化——质量门禁、测试基础设施、CI、Lint |

## 需求文件格式 (backlog.d/)

```markdown
# 修复认证 Token 轮换

Priority: high
Status: ready | blocked | in-progress | done

## 目标
<1 句话——什么结果，不是什么机制>

## 非目标
- <不做的>

## 验收标准
- [ ] 可机械验证的标准（优先用命令而非文字）

## 备注
<上下文、约束、现有方案>
```

## 调查方法

### Archaeologist（代码健康）
复杂度热点、测试缺口、耦合问题、死代码

### Strategist（产品机会）
用户体验摩擦、缺失功能、竞争差距

### Velocity（效率模式）
修复/功能比、变更热点、停滞工作

## 优先级排序

1. **阻塞性 bug** > 一切
2. **安全漏洞** > 功能需求
3. **用户痛点** > 内部改进
4. **高价值/低工作量** > 高价值/高工作量
